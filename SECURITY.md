# Security Policy

## Supported Versions

Security fixes are provided for the latest code on `main`.

## Reporting a Vulnerability

Please report vulnerabilities privately using GitHub Security Advisories for this repository.

- Do not create public issues for suspected vulnerabilities.
- Include reproduction steps, impact, and affected paths/commits.
- If possible, include a minimal proof of concept.

## Response Targets

- Initial acknowledgement: within 3 business days
- Triage decision: within 7 business days
- Patch/mitigation timeline: depends on severity and blast radius

## Secrets and Release Safety

- Never commit credentials or signing material.
- Production macOS releases should use Developer ID signing and notarization.
